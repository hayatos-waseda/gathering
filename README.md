# Gathering 課題

## 概要

7×7のグリッドマップ上で、エージェントを動かして報酬（★）を集め、スコアを競うゲームです。
あなたが実装するのは `agent/action_a.py` の `act()` メソッドだけです。

---

## 環境構築

以下のライブラリが必要です。

```bash
pip install numpy matplotlib pyyaml
```

### 実行

```bash
python3 main.py
```

実行すると `results.gif` が生成されます。最初の100ステップの様子を確認できます。

## ゲームルール

### マップ

```
.......
.##.##.
.#...#.
.......
.#...#.
.##.##.
.......
```

`.` が通路、`#` が壁です。壁は移動・攻撃を遮断します。

### エージェント

4体のエージェントが2チームに分かれて対戦します。

| チーム | エージェント | 開始位置 |
|--------|-------------|---------|
| A（赤） | playerA1 | (0, 0) |
| A（赤） | playerA2 | (6, 0) |
| B（青） | playerB1 | (0, 6) |
| B（青） | playerB2 | (6, 6) |

### 行動

毎ステップ、各エージェントは0〜8の整数を返します。

| action | 内容 |
|--------|------|
| 0 | 上に移動 |
| 1 | 右に移動 |
| 2 | 下に移動 |
| 3 | 左に移動 |
| 4 | 上に攻撃（射程2） |
| 5 | 右に攻撃（射程2） |
| 6 | 下に攻撃（射程2） |
| 7 | 左に攻撃（射程2） |
| 8 | 何もしない |

壁や端に向かって移動しようとした場合は、その場に留まります。

### イベント（★）

各マスには報酬確率 `p` が設定されており、毎ステップその確率で★が出現します。
★があるマスに乗るとスコアを獲得できます。

ただし以下の場合はスコアを獲得できません。
- 複数のエージェントが同じマスにいる場合（★は消滅）
- `broken` 状態の場合

### 攻撃・やられ判定

攻撃が当たったエージェントは `broken` 状態になります。

- `broken` 中は行動できず、座標が `[-1, -1]` になります
- 6ステップ後にマップ上のランダムな位置にリスポーンします
- 攻撃は味方にも当たります

`agent/action_a.py` の `act()` メソッドを実装してください。

```python
# agent/action_a.py

class ActionA:
    def __init__(self, rnd, field_view):
        self.field = field_view
        self.rnd = rnd

    def act(self, pos, a_data, e_data):
        # ここを実装する
        return 
```
実装したら、agent_aにインポートするファイルを変更してください。

### 引数

| 引数 | 型 | 内容 |
|------|----|------|
| `pos` | `[x, y]` | 自分の現在座標 |
| `a_data` | `list` | 味方エージェントの情報リスト |
| `e_data` | `list` | 敵エージェントの情報リスト |

`a_data` と `e_data` は以下の形式の辞書のリストです。

```python
[
    {"pos": [x, y], "status": "active"},  # active または broken
    ...
]
```

`broken` 状態のエージェントの座標は `[-1, -1]` です。

### 返り値

0〜8の整数を返してください。

## 使えるAPI

### FieldView

`self.field` から以下のメソッドが使えます。

#### `get_pos_status(x, y, act) -> int`

座標 `(x, y)` から方向 `act` に移動・攻撃できるかを返します。

- 返り値: `1` なら可能、`0` なら壁またはマップ端

```python
if self.field.get_pos_status(x, y, 0) == 1:
    # 上に移動できる
```

方向の対応は以下の通りです。

| act | 方向 |
|-----|------|
| 0 | 上 |
| 1 | 右 |
| 2 | 下 |
| 3 | 左 |

#### `get_event(x, y) -> int`

座標 `(x, y)` に★があるかを返します。

- 返り値: `1` なら★あり、`0` なら★なし

```python
if self.field.get_event(x, y) == 1:
    # (x, y) に★がある
```

#### `is_path(x, y) -> bool`

座標 `(x, y)` が通路かどうかを返します。

```python
if self.field.is_path(x, y):
    # 通路である
```

#### `grid_size -> int`

マップのサイズを返します（現在は `7`）。

```python
for x in range(self.field.grid_size):
    for y in range(self.field.grid_size):
        ...
```

## ユーティリティ

### A\*（最短経路探索）

`agent/utils/astar.py` にA\*アルゴリズムが用意されています。

```python
from agent.utils.astar import astar

path = astar(self.field, start, goal)
```

| 引数 | 型 | 内容 |
|------|----|------|
| `field_view` | `FieldView` | `self.field` を渡す |
| `start` | `[x, y]` | 開始座標 |
| `goal` | `[x, y]` | 目標座標 |

**返り値**: `start` から `goal` までの座標リスト（`start` と `goal` を含む）

```python
path = astar(self.field, [0, 0], [3, 3])
# → [[0, 0], [0, 1], [0, 2], [0, 3], [1, 3], [2, 3], [3, 3]]
```

経路が見つからない場合は空リスト `[]` を返します。

`path[1]` が次に進むべきマスです。そこへの移動方向（0〜3）への変換は自分で実装してください。
